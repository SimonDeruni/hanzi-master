# 🔴 Issues & Bugs Update Standard

All entries in `docs/ISSUES.md` must follow this standard.

## 📊 Table Structure
Maintain the existing Markdown table. New issues go to the **bottom** of the active list.

## 🚥 Status Labels
- **🔴 NEEDS TO BE SOLVED:** Active issues.
- **✅ FIXED:** Resolved issues.

## 📝 Table Entry Format
`| [YYYY-MM-DD] | **[Title]**: [Technical Description] | [🔴 NEEDS TO BE SOLVED / ✅ FIXED] |`

## ⚖️ Severity Rules
- Always identify if an issue is **Security**, **Performance**, or **Logic** related in the description.
