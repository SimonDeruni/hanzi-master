# 🚀 Feature Manifest Update Standard

All updates to `docs/FEATURE_MANIFEST.md` must follow this standard.

## 🏛️ Section Integrity
- **(CURRENT)**: Only features that are 100% merged and functional in the `lib/` folder.
- **Planned & Future**: Features that are brainstormed but not yet implemented.

## ✍️ Pedagogical Tone
- Describe features in plain English so the non-native founder can understand.
- Use bullet points for sub-features.
- **Formatting:** Keep headers (H2) for major domains and H3 for sub-domains.
