# 🛑 Anti-Patterns Standard

All updates to `docs/ANTI_PATTERNS.md` must follow this standard.

## 📅 When to Update
- **Audit Failure:** If an audit identifies a technical approach that failed.
- **Lesson Learned:** If a bug fix reveals a "Trap" that future agents should avoid.

## 📝 Entry Template
```markdown
### [Context Name]
- **Trap:** [The rejected approach]
- **Why:** [Why it failed/janked/was insecure]
- **Better Way:** [The approved alternative]
```
