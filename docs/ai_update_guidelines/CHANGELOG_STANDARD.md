# 📜 Changelog Update Standard

All entries in `docs/CHANGELOG.md` must follow this standard.

## 📅 Timeline Rule
- **Reverse Chronological:** The newest entry MUST always be at the top.
- **Date Format:** `[YYYY-MM-DD]`

## 🏷️ Technical Labels
Use these exact tags for the [TYPE] section:
- `[✨ ADDED]`: New features or widgets.
- `[🔧 FIXED]`: Bug fixes or logic corrections.
- `[⚖️ AUDITED]`: Technical reviews and findings.
- `[🧪 TESTED]`: Implementation of new tests.
- `[🔄 SYNC]`: Documentation or organizational updates.

## 🔒 4. The Session Seal (Mandatory)
Every session MUST conclude with a final entry labeled `[🏁 CONCLUSION]`.
- **Claims Check:** Confirm all `[🔒 LOCKED]` claims in `SESSION_STATE.md` were removed.
- **File Manifest:** List every file touched in this session.
- **Hygiene Rating:** Rate your own performance based on the AI_PROTOCOL (1-10).
